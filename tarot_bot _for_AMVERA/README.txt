├── bot.py                    # Главный файл запуска
├── config.py                 # Конфигурация и переменные окружения
├── requirements.txt          # Список Python-зависимостей
├── .env                      # Секретные переменные (токены, ключи)
├── .env.example              # Шаблон для .env
├── tarot_bot.db              # База данных SQLite (создаётся автоматически)
│
├── handlers/                 # Обработчики команд и событий
│   ├── __init__.py
│   ├── start.py             # Онбординг и главное меню
│   ├── spreads.py           # Логика раскладов Таро
│   ├── payments.py          # Обработка платежей (ЮKassa)
│   └── history.py           # История раскладов пользователя
│
├── db/                       # Работа с базой данных
│   ├── __init__.py
│   └── database.py          # Класс Database со всеми методами
│
├── keyboards/                # Inline-клавиатуры
│   ├── __init__.py
│   └── keyboards.py         # Все кнопки и меню бота
│
└── utils/                    # Вспомогательные утилиты
    ├── __init__.py

Описание файлов

🔹 bot.py — Главный файл запуска
Назначение: Точка входа в приложение. Отвечает за:
Инициализацию базы данных
Регистрацию всех обработчиков команд
Настройку JobQueue для планировщика задач
Запуск бота в режиме polling
Глобальную обработку ошибок
Ключевые компоненты:
Класс TarotBot — объединяет все обработчики
build_application() — собирает приложение из компонентов
error_handler() — отправляет алерты админу при ошибках
post_init() — инициализирует БД при старте

🔹 config.py — Конфигурация
Назначение: Хранит все настройки приложения и загружает переменные из .env.
Что содержит:
BOT_TOKEN — токен бота от @BotFather
YOOKASSA_SHOP_ID, YOOKASSA_SECRET_KEY — ключи для платежей
ADMIN_CHAT_ID — ID админа для алертов
PRICES — словарь с ценами пакетов (в рублях)
PACKAGES — описание пакетов (количество раскладов)
WELCOME_BONUS — бонус для новых пользователей
Как работает:
Использует библиотеку python-dotenv для чтения .env файла.

🔹 requirements.txt — Зависимости Python
Назначение: Список всех библиотек, необходимых для работы бота

🔹 .env — Секретные переменные
Назначение: Хранит конфиденциальные данные (токены, ключи). Не коммитится в Git!

🔹 handlers/start.py — Онбординг и главное меню
Назначение: Обрабатывает первый запуск бота и навигацию по меню.
Что делает:
/start — приветствие нового пользователя
Запрашивает имя пользователя
Запрашивает цель (любовь/работа/саморазвитие)
Обрабатывает реферальные ссылки (?start=ref_123)
Показывает главное меню с раскладами
Обрабатывает opt-in для ежедневной карты
Ключевые методы:
start() — точка входа для новых пользователей
ask_name(), ask_goal() — шаги онбординга
show_main_menu() — отображение главного меню
daily_optin() — подписка на ежедневные пуши

🔹 handlers/spreads.py — Логика раскладов
Назначение: Обрабатывает все типы раскладов Таро.
Что делает:
Проверяет лимиты бесплатных раскладов (Карта дня: 1/день, Да/Нет: 5/день)
Проверяет баланс для платных раскладов
Генерирует случайные карты
Анимированно раскрывает карты (1 секунда между картами)
Сохраняет расклады в БД
Обновляет уровень пользователя
Обрабатывает команды /balance и /referral
Типы раскладов:
Карта дня — 1 карта, бесплатно (лимит 1/день)
Да/Нет — 1 карта, бесплатно (лимит 5/день)
Прошлое-Настоящее-Будущее — 3 карты, платно
Расклад на отношения — 7 карт, платно
Кельтский крест — 10 карт, платно (с уникальными интерпретациями)
Ключевые методы:
handle_spread() — основной обработчик раскладов
balance_command() — показывает баланс и статистику
referral_command() — генерирует реферальную ссылку

🔹 handlers/payments.py — Обработка платежей
Назначение: Интеграция с ЮKassa для приёма платежей в рублях.
Что делает:
Показывает меню с пакетами раскладов
Создаёт платёж в ЮKassa
Генерирует ссылку для оплаты картой
Проверяет статус платежа каждые 10 секунд (через JobQueue)
Зачисляет расклады после успешной оплаты
Отправляет уведомление пользователю
Пакеты:
1 расклад — 59 ₽
5 раскладов — 249 ₽ (экономия 16%)
10 раскладов — 449 ₽ (экономия 24%)
Безлимит на 30 дней — 699 ₽ (экономия 35%)
Ключевые методы:
show_buy_menu() — отображает меню покупки
send_invoice() — создаёт платёж в ЮKassa
check_payment_status() — проверяет статус платежа

🔹 handlers/history.py — История раскладов
Назначение: Показывает пользователю его прошлые расклады.
Что делает:
Обрабатывает команду /history
Обрабатывает кнопку "📜 История раскладов"
Показывает последние 10 раскладов с датами и картами
Форматирует историю в читаемом виде
Ключевые методы:
show_history() — для кнопки (edit_text)
history_command() — для команды (reply_text)

🔹 db/database.py — Работа с базой данных
Назначение: Все операции с SQLite через aiosqlite.
Таблицы:
users — информация о пользователях
user_id, username, first_name
balance — количество доступных раскладов
is_unlimited, unlimited_until — статус безлимита
total_spreads — общее количество раскладов
level — уровень пользователя (Новичок/Искатель/Маг/Верховный Жрец)
referred_by — ID пригласившего пользователя
welcome_bonus_used — использован ли welcome-бонус
spreads — история раскладов
user_id, spread_type, cards (JSON), positions (JSON)
created_at — дата создания
payments — история платежей
user_id, package_type, spreads_added, amount_stars
Ключевые методы:
get_or_create_user() — создаёт пользователя или возвращает существующего
get_balance(), use_spread() — работа с балансом
check_daily_spread() — проверяет лимиты бесплатных раскладов
process_referral() — обрабатывает реферальную систему
save_spread(), get_history() — работа с историей раскладов
update_level() — обновляет уровень пользователя

🔹 keyboards/keyboards.py — Inline-клавиатуры
Назначение: Все кнопки и меню бота.
Функции:
get_main_menu() — главное меню с раскладами (динамически показывает оставшиеся лимиты)
get_onboarding_goal() — выбор цели при онбординге
get_buy_menu() — меню покупки пакетов
get_after_spread_menu() — меню после расклада
get_back_to_main() — кнопка "Назад в меню"
get_daily_card_optin() — подписка на ежедневную карту
Особенность:
get_main_menu() принимает параметры remaining_card и remaining_yesno для динамического отображения лимитов:

get_main_menu(remaining_card=1, remaining_yesno=5)
# Покажет: "☀️ Карта дня (бесплатно)" или "☀️ Карта дня (завтра)"

🔹 utils/tarot_deck.py — Колода Таро
Назначение: Содержит все 22 Старших Аркана с описаниями.
Структура карты:

"1": {
    "name": "Маг",
    "emoji": "🎩",
    "upright": "Общее значение (прямая карта)",
    "reversed": "Общее значение (перевёрнутая карта)",
    "celtic": {
        "upright": "Уникальный совет для Кельтского креста (прямая)",
        "reversed": "Уникальный совет для Кельтского креста (перевёрнутая)"
    },
    "yes_no": "Да"
}

Словарь SPREADS:
Описывает все типы раскладов:
name — название
cards — количество карт
positions — список позиций (например, ["Прошлое", "Настоящее", "Будущее"])
paid — платный или бесплатный
description — описание для пользователя
🔹 utils/formatters.py — Форматирование сообщений
Назначение: Превращает данные в красивый HTML-текст для Telegram.
Функции:
get_random_cards(count) — генерирует случайные карты со случайной ориентацией
format_card_with_position() — форматирует одну карту с позицией
Для Кельтского креста: компактный формат с уникальными советами
Для остальных: стандартный формат
format_full_spread() — форматирует полный расклад с заголовком
get_progress_bar() — создаёт прогресс-бар из эмодзи (🔮⚪)
format_history_item() — форматирует элемент истории
get_level_info() — возвращает название уровня по количеству раскладов

🐍 Работа с виртуальным окружением (venv)
Виртуальное окружение изолирует зависимости проекта от системного Python. Это предотвращает конфликты версий библиотек.
Создание venv

macOS/Linux:
    cd /path/to/tarot_bot
    python3.11 -m venv venv
    source venv/bin/activate

Windows:
    cd C:\path\to\tarot_bot
    python -m venv venv
    venv\Scripts\activate.bat

Удаление venv:

macOS/Linux:
    rm -rf venv

Windows:
    rmdir /s venv

----------------------------------

Установка зависимостей через pip3:

Базовая установка
1. Активируйте venv (см. выше)
2. Обновите pip:
    pip install --upgrade pip
3. Установите все зависимости из requirements.txt
pip install python-telegram-bot==20.7 aiosqlite python-dotenv aiohttp yookassa
pip install "python-telegram-bot[job-queue]"
